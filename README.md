# The Chester Guinness Index

A premium, map-based Guinness pub review platform for Chester. Score pubs out of 50, manage everything from your phone, and auto-generate social media posts from each review.

---

## Features

- **Interactive Google Map** with custom gold markers, dark styling, click-to-reveal info windows
- **Ranked pub list** synced to map, filterable by area, score, and price
- **Scoring system** — 5 categories × 10 points = 50 max, with named rating tiers
- **Mobile-first admin panel** — add pubs, write reviews with live score display, manage social drafts
- **Auto social posts** — generated by database trigger on every new review
- **Leaderboard** — overall, best taste, best value, best pour, most recent

## Setup

### 1. Clone and install

```bash
git clone https://github.com/your-repo/chester-guinness-index
cd chester-guinness-index
npm install
```

### 2. Environment variables

```bash
cp .env.example .env.local
```

Fill in:

| Variable                         | Where to get it                          |
|----------------------------------|------------------------------------------|
| `NEXT_PUBLIC_SUPABASE_URL`       | Supabase project → Settings → API        |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY`  | Supabase project → Settings → API        |
| `SUPABASE_SERVICE_ROLE_KEY`      | Supabase project → Settings → API        |
| `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY`| Google Cloud Console → Maps JS API key   |

### 3. Supabase setup

In your Supabase project SQL editor, run:

```sql
-- Run the full schema migration
\i supabase/migrations/001_initial.sql

-- Optional: seed with 15 Chester pubs and sample reviews
\i supabase/seed.sql
```

Or paste the file contents directly into the SQL editor.

**Google Maps API key restrictions** (recommended):
- Application restrictions: HTTP referrers
- Add: `localhost:3000/*`, `yourdomain.com/*`
- API restrictions: Maps JavaScript API only

### 4. Create admin user

In Supabase → Authentication → Users → Add user:
- Email: your admin email
- Password: strong password

### 5. Run development server

```bash
npm run dev
```

Visit [http://localhost:3000](http://localhost:3000)

## VPS Deployment

### Prerequisites

- Ubuntu 22.04+ VPS (1GB RAM minimum)
- Node.js 20+
- Nginx
- PM2

### 1. Server setup

```bash
# Install Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Install PM2
sudo npm install -g pm2

# Install Nginx
sudo apt install -y nginx
```

### 2. Deploy application

```bash
# Clone repo on server
git clone https://github.com/your-repo/chester-guinness-index /var/www/cgi
cd /var/www/cgi

# Install dependencies
npm ci --production=false

# Create .env.local with production values
nano .env.local

# Build
npm run build

# Start with PM2
pm2 start npm --name "chester-guinness-index" -- start
pm2 save
pm2 startup
```

### 3. Nginx configuration

```nginx
# /etc/nginx/sites-available/chester-guinness-index
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/chester-guinness-index /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 4. SSL (Let's Encrypt)

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

### 5. Deploy updates

```bash
cd /var/www/cgi
git pull
npm ci --production=false
npm run build
pm2 restart chester-guinness-index
```

## Database Schema Overview

```
pubs            — pub registry with current_score (auto-updated by trigger)
reviews         — scored reviews; total_score is a generated column
pub_images      — gallery images (Supabase Storage)
social_posts    — auto-generated from reviews via DB trigger
```

All tables have Row Level Security:
- Public: `SELECT` on pubs, reviews, pub_images
- Authenticated (admin): full CRUD

## Admin Usage

1. Visit `/login` — sign in with your Supabase admin credentials
2. `/admin` — dashboard with stats and quick actions
3. `/admin/pubs` — add/edit pubs (name, coordinates, price, area)
4. `/admin/reviews` — add reviews with live score display and tier badge
5. `/admin/social` — copy/edit/mark-as-posted the auto-generated social drafts

> **Tip:** The review form is optimized for mobile. Open `/admin/reviews?action=new` on your phone to score a pint on the spot.
